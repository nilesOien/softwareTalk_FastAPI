The uv package manager

With uv the process is a bit different. Of course, uv has to be installed for this to work.

First, we can optionally select a version of python (3.13 here) using the uv python pin command like so : 


 $ uv python pin 3.13

Then using uv we intialize a project, set up a virtual environment, set up a file requirements.in that lists dependencies (sunpy here) and install the dependencies, like so : 


 $ cd uv/
 $ cat requirements.in
   sunpy[all]
 $ uv init --name uv_example --description "An example of uv" --bare .
 $ uv run python --version
   Python 3.13.7
 $ uv add -r requirements.in
That install takes about 3 seconds on my machine.

Then, using uv, we run the program : 


 $ uv run ../sunpyPrintVersion.py
 7.0.3
Notice the difference between uv and pip. While pip relies on entering an environment, uv relies on running things through the uv run command. Also the later version of python has resulted in a later version of sunpy being installed.

The uv package manager is used for the remainder of this discussion.
